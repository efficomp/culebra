culebra.fitness_function.cooperative.KappaNumFeatsC class¶

class KappaNumFeatsC(training_data: Dataset, test_data: Dataset | None = None, test_prop: float | None = None, classifier: ClassifierMixin = SVC())¶

Construct a fitness function.

If test_data are provided, the whole training_data are used to train the classifier. Otherwise, if test_prop is provided, training_data are split (stratified) into training and test data. Finally, if both test_data and test_prop are omitted, training_data are also used to test.

Parameters:

  • training_data (Dataset) – The training dataset

  • test_data (Dataset, optional) – The test dataset, defaults to None

  • test_prop (float, optional) – A real value in (0, 1) or None. Defaults to None

  • classifier (SVC with RBF kernels, optional) – The classifier

More information about this fitness function can be found in [Gonzalez2021].

KappaNumFeatsC.evaluate(sol: Solution, index: int | None = None, representatives: Sequence[Solution] | None = None) → Tuple[float, ...]¶

Evaluate a solution.

This fitness function assumes that:

  • representatives[0]: Codes the SVC hyperparameters (C and gamma). Thus, it is an instance of culebra.solution.parameter_optimization.Solution

  • representatives[1:]: The remaining solutions code the features selected, each solution a different range of features. All of them are instances of culebra.solution.feature_selection.Solution

Parameters:

  • sol (Solution) – Solution to be evaluated.

  • index (int) – Index where sol should be inserted in the representatives sequence to form a complete solution for the problem

  • representatives (Sequence of Solution, ignored) – Representative solutions of each species being optimized

Returns:

The fitness of sol

Return type:

tuple of py:class:float
