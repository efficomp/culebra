fitness_function.cooperative.KappaNumFeatsC class¶

class KappaNumFeatsC(training_data: Dataset, test_data: Optional[Dataset] = None, test_prop: Optional[float] = None, classifier: ClassifierMixin = SVC())¶

Construct a fitness function.

If test_data are provided, the whole training_data are used to train the classifier. Otherwise, if test_prop is provided, training_data are split (stratified) into training and test data. Finally, if both test_data and test_prop are omitted, training_data are also used to test.

Parameters:

  • training_data (Dataset) – The training dataset

  • test_data (Dataset, optional) – The test dataset, defaults to None

  • test_prop (float, optional) – A real value in (0, 1) or None. Defaults to None

  • classifier (SVC with RBF kernels, optional) – The classifier

More information about this fitness function can be found in [Gonzalez2021].

[Gonzalez2021]

J. González, J. Ortega, J. J. Escobar, M. Damas. A lexicographic cooperative co-evolutionary approach for feature selection. Neurocomputing, 463:59-76, 2021. https://doi.org/10.1016/j.neucom.2021.08.003.

Methods¶

KappaNumFeatsC.evaluate(ind: Individual, index: Optional[int] = None, representatives: Optional[Sequence[Individual]] = None) → Tuple[float, ...]¶

Evaluate an individual.

This fitness function assumes that:

  • representatives[0]: Codes the SVC hyperparameters (C and gamma). Thus, it is an instance of genotype.classifier_optimization.Individual

  • representatives[1:]: The remaining individuals code the features selected, each individual a different range of features. All of them are instances of genotype.feature_selection.Individual

Parameters:

  • ind (Individual) – Individual to be evaluated.

  • index (int) – Index where ind should be inserted in the representatives sequence to form a complete solution for the problem

  • representatives (Sequence of Individual, ignored) – Representative individuals of each species being optimized

Returns:

The fitness of ind

Return type:

tuple of py:class:float
